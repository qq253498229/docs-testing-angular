= 简介

学习目标

- 自动化软件测试的好处
- 为什么测试很难学习
- 实现和测试之间的区别
- 建立稳定的测试实践

大多数Web开发人员在他们的职业生涯中都会遇到自动化测试。他们喜欢编写代码来审查Web应用程序并对其进行严格测试的想法。作为Web开发人员，作为商业人士，我们希望知道该网站是否适用于用户，我们的客户。

该网站是否允许用户完成其任务？在引入新功能或重构内部之后，该网站是否仍然可用？该网站如何对使用错误或系统故障做出反应？测试可以回答这些问题。

我相信自动化测试的好处很容易理解。开发人员希望能够安心睡觉并确信他们的应用程序正常运行。此外，测试有助于开发人员编写更好的软件。更健壮、更易于理解、更易于维护的软件。

与此形成鲜明对比的是，我遇到的只有少数Web开发人员拥有稳定的测试实践。只有少数人觉得编写测试 _容易_，更不要说 _享受_ 编写测试了。这项任务被视为一项琐事或麻烦。

通常，个别开发人员会因缺乏测试而受到指责。声称开发人员只是太无知或懒惰，不愿意编写测试，这是过于简单化和有害的。如果测试具有不容置疑的价值，那么我们需要考察为什么开发人员会避免它，同时又确信其好处。测试应该易于、直接和常见。

如果您在编写测试方面遇到困难，那不是您的错或缺陷。由于测试软件本质上是复杂且困难的，我们都在努力挣扎。

首先，编写自动化测试需要与编写实现代码不同的思维方式。实施功能意味着构建结构-测试意味着试图推翻它。

您会尝试在自己的工作中找到弱点和漏洞。您会考虑所有可能的情况，并用“如果”问题来干扰您的代码。乍一看似乎令人沮丧，但这是改进代码的宝贵策略。

其次，测试具有陡峭的学习曲线。如果将测试视为工具，则不像螺丝刀或电钻。相反，它类似于拖拉机或挖掘机。操作这些机器需要培训。并且需要经验才能准确安全地应用它们。

这是为了鼓励您。开始测试很难，但随着更多的实践，它变得越来越容易。本指南的目标是使您有能力每天编写测试，以覆盖您的Angular应用程序的重要功能。
